Everything you need to ship identity

Production-ready features, built-in from day one.

🔐

ECDSA P-256 Signing

Every identity packet is cryptographically signed before leaving the server. Verify offline with just the public key.

📡

NFC Tap Sharing

Three modes: URL (browser), VCARD (contacts app), IDENTITY_PACKET (app-to-app signed JSON). Zero friction.

🏢

Multi-Tenant RBAC

Tenant isolation with four roles: SUPER_ADMIN, ADMIN, MEMBER, VIEWER. Per-tenant API keys with scoped permissions.

⚡

JWT Edge Middleware

Token validation runs at the CDN edge before your app code. Built-in rate limiting per IP with configurable windows.

📱

QR Code + vCard

Generate QR PNG or download a vCard 3.0 from any profile. Works with every contacts app on every platform.

🩺

Health Check API

GET /api/health returns DB latency, env validation status, and uptime. Returns 200 / 207 / 503.

🐳

Docker + Compose

One-command local stack with Postgres. GHCR image auto-built and Trivy-scanned on every push to main.

🚀

Netlify Deploy

One-command deploy via pnpm setup:netlify. Env vars synced automatically. Preview deploys on every PR.

🎛️

Admin Dashboard

User management, platform statistics, and full audit log — all protected behind the ADMIN role gate.

Quick Start

Running locally in 3 commands. No OpenSSL required.

# 1. Clone & install
$ git clone https://github.com/FTHTrading/Vs-Identity-os.git && cd Vs-Identity-os
$ pnpm install
 
# 2. Auto-generate .env (ECDSA keys + JWT secrets, no openssl)
$ pnpm setup
 
# 3. Start Postgres + run migrations + launch dev server
$ docker compose up -d postgres
$ pnpm db:migrate && pnpm db:seed && pnpm dev
 
# Optional: one-command deploy to Netlify
$ pnpm setup:netlify

Health endpoint
GET /api/health → JSON with DB latency, env status, uptime
